    12→## Architecture Decisions (Phase 1)
    13→- **Stack**: Next.js 16 App Router, TypeScript, Tailwind v4, shadcn/ui (New York), Prisma (SQLite), Recharts, Zustand.
    14→- **Single route**: only `/` is user-visible. The app is a client-side SPA-style shell with a sidebar; modules render as sections driven by a Zustand store (`activeModule`).
    15→- **Theme**: emerald/teal primary on a slate-neutral base (NO indigo/blue). Light + dark mode via next-themes.
    16→- **Dynamic form engine**: forms are defined as JSON schemas stored in DB (`FormSchema` model) and rendered by a generic `<DynamicForm>` React component. Every "Add X" modal uses this engine. A Form Builder lets admins create/edit schemas.
    17→- **APIs**: all under `/api/*` using route handlers + Prisma. Single default tenant (`TENANT_ID` constant) for Phase 1.
    18→- **Workflows**: `Workflow` + `WorkflowStep` models; `WorkflowInstance` tracks each request's progress. Leave/asset/etc. requests flow through workflows.

    97→### API conventions (MANDATORY)
    98→- Every model has `tenantId`. Always call `const tenantId = await ensureTenant()` first in handlers.
    99→- LIST endpoints: `GET /api/<resource>` → returns `{ items: [...] }`
   100→- CREATE: `POST /api/<resource>` with JSON body → returns created object
   101→- UPDATE: `PATCH /api/<resource>/<id>` → returns updated object
   102→- DELETE: `DELETE /api/<resource>/<id>` → returns `{ ok: true }`
   103→- PICKER endpoints: `GET /api/<resource>/picker` → returns `{ items: [{label, value}] }` (used by dynamic form pickers)
   104→- Use `NextRequest`/`Request` from next. Import `db` from `@/lib/db`.
   105→
   106→### Dynamic form usage pattern
   107→```tsx
   108→import { DynamicForm } from "@/components/dynamic-form/dynamic-form"
   109→import { getDefaultSchema } from "@/lib/form-schemas"
   110→const schema = getDefaultSchema("employee")!
   111→<DynamicForm schema={schema} initialValues={row} onSubmit={async (v) => { await fetch('/api/employees', {method:'POST', body: JSON.stringify(v)}) }} onCancel={...} />
   112→```
   113→
   114→### UI patterns
   115→- Every module component is a named export: `export function XxxModule()`
   116→- Use `PageHeader` at top, `ListToolbar` for search+add, `DataTable` for lists, `Dialog`/`Sheet` for add/edit forms with `DynamicForm` inside.
   117→- Wrap data fetching in `useEffect` + `useState` (or react-query). Show `toast.success`/`toast.error` on actions.
   118→- Use `StatusBadge` for status fields. Use emerald accents, never indigo/blue.
