    93‚Üí## 1.3 Tenant Data Isolation
    94‚Üí
    95‚ÜíIndustry-grade HRMS must support strong tenant separation.
    96‚Üí
    97‚Üí### Options
    98‚Üí
    99‚ÜíSingle database, shared schema with `TenantId`.
   100‚Üí
   101‚ÜíSingle database, separate schema per tenant.
   102‚Üí
   103‚ÜíSeparate database per tenant.
   104‚Üí
   105‚ÜíHybrid model: shared DB for small clients, dedicated DB for enterprise clients.
   106‚Üí
   107‚Üí### Required Rules
   108‚Üí
   109‚ÜíEvery table should have `TenantId`.
   110‚Üí
   111‚ÜíEvery API must filter by `TenantId`.
   112‚Üí
   113‚ÜíUsers from one company must never access another company‚Äôs data.
   114‚Üí
   115‚ÜíFiles should be stored tenant-wise.
   116‚Üí
   117‚ÜíAudit logs should include tenant/company/user/IP/device.
   118‚Üí
   119‚ÜíBackups should support tenant-level restore.
