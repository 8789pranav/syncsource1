Yes, onboarding should be a **full workflow engine**, not just “create employee + upload documents.” In industry HRMS, onboarding covers **pre-boarding, candidate-to-employee conversion, dynamic forms, document collection, e-signature, task checklists, IT/admin/payroll/manager tasks, Kanban tracking, approvals, reminders, and employee activation**. Zoho People’s onboarding service includes invite sending, e-signature files, checklists, candidate-to-employee conversion, customizable flows/forms, permissions, custom forms, and automation; Keka positions onboarding around digital offer letters, pre-boarding tasks, day-one checklists, and integrations with payroll, attendance, and asset management. ([Zoho Corporation][1])

# 2. Main Concept

Onboarding should have 3 major parts:

```text
1. Onboarding Workflow
   Defines the complete process.

2. Onboarding Instance
   Actual onboarding case for one candidate/employee.

3. Onboarding Tasks
   Tasks assigned to HR, candidate, manager, IT, admin, payroll, finance, etc.
```

Example:

```text
Workflow: India Full-Time Employee Onboarding

Candidate: Rahul Sharma

Onboarding Instance:
Rahul Sharma - India Full-Time Onboarding

Tasks:
- Candidate fills joining form
- Candidate uploads PAN/Aadhaar
- HR verifies documents
- IT creates email account
- Admin assigns laptop
- Payroll collects bank details
- Manager schedules introduction
```

# 3. Onboarding Stages

Keka describes onboarding in stages such as pre-offer, offer release, post-offer acceptance, pre-joining, joining day, and post-joining; that is a good industry model to follow. ([Keka HR & Payroll Software][2])

Use these stages:

```text
1. Pre-Offer
2. Offer Release
3. Offer Acceptance
4. Pre-Joining / Preboarding
5. Joining Day
6. First Week
7. First 30 Days
8. Probation / Confirmation Handover
9. Completed
```

You can also keep a shorter version:

```text
Preboarding → Joining → Post-Joining → Completed
```

But for industry-level HRMS, the detailed version is better.

# 14. Workflow Fallback Priority

When onboarding starts, system should pick workflow like this:

```text
1. Manually selected workflow
2. Candidate-specific workflow
3. Department + location + employee type workflow
4. Department workflow
5. Location workflow
6. Entity workflow
7. Tenant default workflow
```

If multiple workflows match, highest priority wins.

# 16. Due Date Rule

Each task should support dynamic due dates.

```text
On initiation date + X days
Before joining date - X days
On joining date
After joining date + X days
After offer acceptance + X days
After previous task completion + X days
Specific date
Manual date
```

Examples:

```text
Upload documents = Before joining date - 5 days
IT email creation = Before joining date - 2 days
Welcome meeting = On joining date
Manager feedback = After joining date + 7 days
30-day check-in = After joining date + 30 days
```

# 17. Task Dependency

Industry-level onboarding needs dependency.

```text
Task A must complete before Task B starts.
Task B is blocked until Task A completes.
Stage cannot complete until all mandatory tasks are complete.
```

Example:

```text
Candidate uploads bank details
       ↓
Payroll verifies bank details
       ↓
Payroll profile is activated
```

# 30. Onboarding Initialization Flow

This is how onboarding should start.

## Case 1: From Recruitment

```text
1. Candidate marked as selected.
2. HR clicks “Initiate Onboarding.”
3. System pulls candidate data from ATS.
4. HR selects onboarding workflow.
5. System generates onboarding instance.
6. System creates stages and tasks.
7. Candidate portal invite is sent.
8. Candidate completes profile and documents.
9. HR verifies documents.
10. Offer / appointment process completed.
11. IT/admin/payroll tasks start automatically.
12. Candidate joins.
13. HR marks “Joined.”
14. Candidate converts to employee.
15. Employee code and login are activated.
```

## Case 2: Manual HR Initiation

```text
1. HR opens Initiate Onboarding.
2. HR enters candidate/new hire data.
3. HR selects entity, department, joining date, manager.
4. System auto-suggests workflow.
5. HR reviews tasks and due dates.
6. HR sends candidate invite.
7. Onboarding instance starts.
```

## Case 3: Bulk Onboarding

```text
1. HR downloads Excel template.
2. HR uploads candidate list.
3. System validates data.
4. HR maps workflow by entity/department/employee type.
5. System creates onboarding cases in bulk.
6. Invites are sent in bulk.
7. HR tracks all candidates in Kanban.
```

# 31. Auto Task Creation Logic

When onboarding starts:

```text
1. Get candidate entity, location, department, designation, employee type.
2. Find matching onboarding workflow.
3. Create onboarding instance.
4. Create stages from workflow.
5. Create tasks from task templates.
6. Calculate due dates using joining date and initiation date.
7. Assign task owners dynamically.
8. Send email/in-app notifications.
9. Show case in Kanban.
```

# 32. Auto Stage Movement Logic

```text
If candidate submits profile form:
Move from Candidate Form Pending → Documents Pending

If all documents uploaded:
Move from Documents Pending → HR Verification

If HR verifies documents:
Move from HR Verification → IT/Admin/Payroll Setup

If all mandatory setup tasks completed:
Move to Joining Day

If HR marks joined:
Move to Post-Joining

If post-joining tasks completed:
Move to Completed
```

```text
Cannot send invite without candidate email.
Cannot mark document complete without mandatory documents.
Cannot convert to employee without mandatory profile fields.
Cannot activate payroll without salary structure and bank details.
```

# 50. Final Recommended Onboarding Architecture

Build it like this:

```text
Onboarding Workflow
        ↓
Stages / Kanban Columns
        ↓
Task Templates
        ↓
Forms + Documents + Approvals
        ↓
Onboarding Instance for Candidate
        ↓
Candidate Portal + HR Kanban
        ↓
IT/Admin/Payroll/Manager Tasks
        ↓
Employee Conversion
        ↓
Leave/Attendance/Payroll/Asset/Profile Setup
        ↓
Post-Joining Tasks
        ↓
Completed Onboarding
```
