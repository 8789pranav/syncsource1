## 1.3 Tenant Data Isolation

Industry-grade HRMS must support strong tenant separation.

### Options

Single database, shared schema with `TenantId`.

Single database, separate schema per tenant.

Separate database per tenant.

Hybrid model: shared DB for small clients, dedicated DB for enterprise clients.

### Required Rules

Every table should have `TenantId`.

Every API must filter by `TenantId`.

Users from one company must never access another company’s data.

Files should be stored tenant-wise.

Audit logs should include tenant/company/user/IP/device.

Backups should support tenant-level restore.
